PASSO A PASSO PARA AUTENTICAÇÃO VIA SSH NO GITHUB
================================================

1. Verifique se você já tem uma chave SSH:
   - Abra o Git Bash ou PowerShell
   - Execute:
     ls ~/.ssh
   - Procure por arquivos como id_rsa, id_rsa.pub, id_ed25519 ou id_ed25519.pub

2. Gere uma nova chave SSH (se ainda não tiver):
   - Execute:
     ssh-keygen -t ed25519 -C "seu.email@example.com"
   - Se o sistema não suportar ed25519, use:
     ssh-keygen -t rsa -b 4096 -C "seu.email@example.com"
   - Aceite o local padrão e, se quiser, defina uma senha para a chave

3. Adicione a chave ao agente SSH:
   - Inicie o agente SSH:
     eval $(ssh-agent -s)
   - Adicione a chave privada:
     ssh-add ~/.ssh/id_ed25519
   - Se usou RSA:
     ssh-add ~/.ssh/id_rsa

4. Copie a chave pública:
   - Use um dos comandos abaixo:
     cat ~/.ssh/id_ed25519.pub | clip
   - Ou, para RSA:
     cat ~/.ssh/id_rsa.pub | clip
   - O conteúdo agora está na área de transferência

5. Adicione a chave no GitHub:
   - Acesse github.com e faça login
   - Vá em Settings > SSH and GPG keys
   - Clique em "New SSH key"
   - Cole a chave pública no campo "Key"
   - Dê um título descritivo e salve

6. Teste a conexão SSH:
   - Execute:
     ssh -T git@github.com
   - Confirme a autenticação com "yes" na primeira vez
   - Deve retornar algo como:
     "Hi usuario! You've successfully authenticated, but GitHub does not provide shell access."

7. Use a URL SSH no repositório:
   - Verifique ou altere o remoto para SSH:
     git remote -v
   - Se necessário, configure o remoto SSH:
     git remote set-url origin git@github.com:usuario/repositorio.git

8. Faça push/pull normalmente:
   - git push origin main
   - git pull origin main

DICAS IMPORTANTES
=================
- Não compartilhe sua chave privada (~/.ssh/id_ed25519 ou ~/.ssh/id_rsa).
- Mantenha o agente SSH ativo para evitar reinserção de senha.
- Use chave com senha para mais segurança.
